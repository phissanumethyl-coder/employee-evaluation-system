# ระบบประเมินพนักงานใหม่ (รายสัปดาห์) + HR Dashboard

ระบบประเมินพนักงานใหม่ฝ่ายขาย Digital Marketing 8 สาขา (ภาคอีสาน)
เชื่อม Firebase Firestore · ผู้จัดการประเมินรายสัปดาห์ · HR ดูภาพรวม + ออก PDF ทางการ

## บทบาทผู้ใช้
- **ผู้จัดการสาขา**: เลือกสาขา + ใส่รหัส เห็นเฉพาะพนักงานสาขาตัวเอง เพิ่มพนักงาน + ประเมิน
- **HR ส่วนกลาง**: ใส่รหัส HR เห็นภาพรวมทุกสาขา สรุปรายสัปดาห์/เดือน ดึง PDF

## การประเมิน
- 4 หัวข้อ: ยอดขาย/เป้าหมาย, ทักษะ Digital Marketing, การเรียนรู้/พัฒนา, ทัศนคติ/ความรับผิดชอบ
- แต่ละหัวข้อเลือก 3 ระดับ: ผ่าน / ควรปรับปรุง / ไม่ผ่าน + เขียนความเห็นแนบทุกข้อ
- **ผู้จัดการเป็นคนตัดสินผลรวมเอง** (ผ่าน/ไม่ผ่าน) ระบบไม่คำนวณอัตโนมัติ
- ผล "ไม่ผ่าน" → ระบบ flag เตือน HR ให้โทรแจ้ง (ไม่ตัดสินยุติงานอัตโนมัติ)

## PDF
ออกแบบฟอร์มประเมินทางการภาษาไทย (ฟอนต์ Sarabun) มีช่องลงนามผู้จัดการ + HR
ใช้แนบเอกสารยุติการทำงานได้

## Deploy ขึ้น GitHub Pages
1. push โค้ดขึ้น repo (branch main)
2. Settings → Pages → Source เลือก **GitHub Actions**
3. รอ build เสร็จ เว็บอยู่ที่ `https://<username>.github.io/employee-evaluation-system/`

> ถ้า repo ชื่ออื่น ต้องแก้ `base` ใน `vite.config.js` ให้ตรง

## ตั้งค่า Firebase (สำคัญ)
1. **Firestore Database** → Create database → region `asia-southeast1`
2. **Authentication → Sign-in method** → เปิด **Anonymous**
3. **Firestore → Rules** → วางเนื้อหาจาก `firestore.rules` → Publish
4. **Authentication → Settings → Authorized domains** → เพิ่ม `<username>.github.io`

## รันบนเครื่อง
```bash
npm install
npm run dev
```

## ข้อควรรู้
- Firebase apiKey ในเว็บ client ไม่ใช่ความลับ ความปลอดภัยจริงมาจาก Firestore Rules
- รหัสผ่านต่อสาขาเป็นการกันเบื้องต้นฝั่ง client เหมาะกับใช้ภายในทีม
- **กฎหมายแรงงาน**: การยุติงานควรตรวจสอบเรื่องบอกกล่าวล่วงหน้า/ค่าชดเชยกับที่ปรึกษากฎหมายก่อน
